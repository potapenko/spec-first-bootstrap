## Optional Browser QA Routing

Use this block only if the project has a browser UI and keeps browser QA files
under `qa/`.

If the task is about browser smoke checks, regression runs, or QA reports:

- read `qa/README.md`
- read `qa/web/AGENTS.run.md`

If the task is about creating or updating browser QA cases:

- read `qa/web/AGENTS.cases.md`
- read `qa/web/create-cases-prompt.md`

If the task changes user-visible browser behavior and this project uses browser
QA, update or add the relevant QA artifacts in the same task.
